# SSPanel-UIM 系统配置要求

## 📋 硬件配置要求

### 最低配置（Minimum Requirements）
- **CPU**: 1 核心
- **内存 (RAM)**: 1GB
- **存储空间**: 10GB
- **网络**: 稳定的互联网连接

### 推荐配置（Recommended Requirements）
- **CPU**: 2 核心或以上
- **内存 (RAM)**: 2GB 或以上
- **存储空间**: 20GB SSD（推荐使用SSD以获得更好的性能）
- **网络**: 稳定的互联网连接，建议带宽 ≥ 10Mbps

### 生产环境建议配置
- **CPU**: 4 核心或以上
- **内存 (RAM)**: 4GB 或以上
- **存储空间**: 50GB+ SSD
- **网络**: 稳定的互联网连接，建议带宽 ≥ 50Mbps

---

## 🖥️ 操作系统要求

### 支持的操作系统

#### 推荐系统（Recommended）
- **Debian 12** ⭐（最推荐）
- **Ubuntu 22.04 LTS**
- **Ubuntu 20.04 LTS**

#### 最低要求系统（Minimum）
- **Debian 11+**
- **CentOS 8+ / RHEL 8+**
- **Rocky Linux 8+**
- **AlmaLinux 8+**

> ⚠️ **注意**: 不推荐使用 CentOS 7 或更早版本，因为缺少必要的软件包支持。

---

## 🔧 软件环境要求

### 必需软件

#### 1. Web 服务器
- **Nginx** (必须)
  - 版本: 1.18+ 或更高
  - **HTTPS 必须启用**（生产环境强制要求）
  - 需要支持反向代理功能

#### 2. PHP 环境
- **PHP 版本**: **8.2 或更高版本**（最低要求 8.2）
  - 推荐: PHP 8.4
  - **强烈推荐启用 OPcache + JIT** 以获得最佳性能

#### 3. 数据库
- **MariaDB 10.11+** 或 **MySQL 8.0+**
  - ⚠️ **必须禁用严格模式**（strict mode）
  - 推荐使用 MariaDB 11.8 LTS
  - 需要支持 InnoDB 存储引擎

#### 4. 缓存服务
- **Redis 7.0+**
  - 用于会话存储、缓存和队列管理
  - 必须运行并正确配置

#### 5. 开发工具
- **Git** - 用于代码版本控制
- **Composer 2.1+** - PHP 依赖管理工具

## 📦 PHP 扩展要求

### 必需扩展（Required Extensions）

根据 `composer.json` 和安装脚本，以下扩展是**必须安装**的：

#### 核心扩展
- ✅ **bcmath** - 高精度数学运算
- ✅ **curl** - HTTP 客户端
- ✅ **fileinfo** - 文件类型检测（PHP 5.3+ 已内置）
- ✅ **json** - JSON 处理（PHP 8.0+ 已内置）
- ✅ **mbstring** - 多字节字符串处理
- ✅ **mysqli** - MySQL 数据库连接
- ✅ **openssl** - SSL/TLS 加密支持
- ✅ **pdo** - 数据库抽象层
- ✅ **posix** - POSIX 系统调用
- ✅ **redis** - Redis 客户端
- ✅ **xml** - XML 处理
- ✅ **yaml** - YAML 配置文件解析
- ✅ **zip** - ZIP 文件处理

#### 推荐扩展
- ✅ **gmp** - 大整数运算（用于加密算法）
- ✅ **bz2** - Bzip2 压缩
- ✅ **gd** - 图像处理（用于验证码等功能）
- ✅ **intl** - 国际化支持
- ✅ **opcache** - PHP 操作码缓存（强烈推荐）
- ✅ **soap** - SOAP 协议支持

### 扩展检查命令


## 🌐 网络要求

### 端口要求

#### 必须开放的端口
- **80** (HTTP) - 用于 Web 访问和 SSL 证书验证
- **443** (HTTPS) - 用于安全的 Web 访问（生产环境必须）

#### 可选端口（根据功能需求）
- **3306** (MySQL/MariaDB) - 数据库端口（建议仅本地访问）
- **6379** (Redis) - 缓存服务端口（建议仅本地访问）
- **22** (SSH) - 服务器管理端口

### 防火墙配置

```bash
# Ubuntu/Debian (UFW)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# CentOS/RHEL (firewalld)
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

## 🔐 安全要求

### SSL/TLS 证书
- **生产环境必须使用 HTTPS**
- 推荐使用 Let's Encrypt 免费证书
- 证书必须有效且未过期

### 文件权限
- Web 根目录: `755` (目录) / `644` (文件)
- 配置文件: `600` 或 `640`（保护敏感信息）
- 存储目录: `755`（可写）

### 数据库安全
- 使用强密码
- 限制数据库用户权限（仅授予必要权限）
- 禁用远程访问（如不需要）

## 📊 性能优化建议

### PHP 配置优化

```ini
; php.ini 推荐配置
memory_limit = 256M
max_execution_time = 300
upload_max_filesize = 50M
post_max_size = 50M
max_input_vars = 5000

; OPcache 配置（强烈推荐）
opcache.enable = 1
opcache.memory_consumption = 128
opcache.interned_strings_buffer = 8
opcache.max_accelerated_files = 10000
opcache.revalidate_freq = 2
opcache.fast_shutdown = 1

; JIT 配置（PHP 8.0+）
opcache.jit_buffer_size = 100M
opcache.jit = tracing
```

### MySQL/MariaDB 优化

```ini
# my.cnf 推荐配置
innodb_buffer_pool_size = 1G  # 根据内存调整（建议为内存的 50-70%）
max_connections = 200
query_cache_size = 64M
tmp_table_size = 64M
max_heap_table_size = 64M
```

### Redis 优化

```conf
# redis.conf 推荐配置
maxmemory 512mb
maxmemory-policy allkeys-lru
save ""  # 如果不需要持久化，可以禁用
```

```bash
# 检查 PHP 版本和扩展
php -v

ss -tlnp | grep -E ':(80|443|3306|6379)'
```
