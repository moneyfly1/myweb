# 节点采集功能部署说明

## 📋 功能概述

本次更新为 SSPanel-UIM 添加了节点自动采集功能，可以从外部 URL 自动采集节点并保存到数据库，大大减少手动配置节点的工作量。

## ✨ 新增功能

1. **节点自动采集**
   - 支持从多个 URL 源采集节点
   - 支持多种协议：vmess, ss, trojan, vless, ssr, hysteria2, tuic
   - 自动解析节点配置并保存到数据库

2. **采集配置管理**
   - 管理采集源 URL 列表
   - 设置采集间隔
   - 启用/禁用定时采集
   - 关键词过滤功能

3. **采集日志**
   - 查看采集历史记录
   - 实时查看采集状态

4. **节点来源标识**
   - 节点列表显示节点来源（采集/手动）
   - 区分采集节点和手动配置节点

## 📦 部署步骤

### 1. 运行数据库迁移

```bash
# 进入项目目录
cd /path/to/SSPanel-UIM-master

# 运行迁移
php xcat Migration
```

这将添加以下字段到 `node` 表：
- `is_collected` - 是否为采集节点
- `source_url` - 节点来源 URL
- `collected_at` - 采集时间

### 2. 检查文件权限

确保以下目录有写入权限：
- `storage/` - 日志和缓存目录

### 3. 配置采集源

1. 登录管理员后台
2. 进入 **节点管理** → **节点采集**
3. 添加采集源 URL（每行一个）
4. 配置采集间隔（建议 3600 秒，即 1 小时）
5. 启用定时采集（可选）
6. 点击 **保存配置**

### 4. 测试采集功能

1. 在采集配置页面点击 **立即采集**
2. 查看采集日志确认是否成功
3. 返回节点列表查看采集到的节点

### 5. 配置定时任务（可选）

如果启用了定时采集，确保 Cron 任务正常运行：

```bash
# 编辑 crontab
crontab -e

# 添加以下行（每 5 分钟执行一次）
*/5 * * * * cd /path/to/SSPanel-UIM-master && php xcat Cron
```

## 🔧 配置说明

### 采集源 URL 格式

支持以下格式的订阅链接：
- Base64 编码的订阅链接
- 直接包含节点链接的文本
- 支持协议：vmess, ss, trojan, vless, ssr, hysteria2, tuic

### 过滤关键词

在节点名称中包含这些关键词的节点将被过滤（不采集）。

### 采集间隔

- 最小间隔：60 秒
- 建议间隔：3600 秒（1 小时）
- 定时采集会在 Cron 任务执行时检查是否到了采集时间

## 📝 使用说明

### 手动触发采集

1. 进入 **节点管理** → **节点采集**
2. 点击 **立即采集** 按钮
3. 等待采集完成（查看日志）

### 查看采集日志

在采集配置页面底部可以查看最近的采集日志，包括：
- 采集时间
- 日志级别（info/success/error）
- 日志消息

### 节点管理

- **采集节点**：自动从外部 URL 采集的节点，标记为蓝色"采集"标签
- **手动节点**：手动配置的节点，标记为绿色"手动"标签
- 采集节点可以编辑部分字段（如名称、倍率等），但核心配置（server, port）建议保持原样

## ⚠️ 注意事项

1. **数据去重**
   - 系统会根据 `server + port + sort` 判断节点是否已存在
   - 已存在的采集节点会被更新，而不是创建新节点

2. **节点更新**
   - 采集节点更新时会保留原有的部分配置（如 traffic_rate, node_class 等）
   - 如果节点名称发生变化，会被更新

3. **性能考虑**
   - 大量节点采集可能耗时较长，建议在低峰期执行
   - 建议采集间隔不要设置过短（至少 60 秒）

4. **错误处理**
   - 如果某个 URL 采集失败，会继续处理其他 URL
   - 解析失败的节点会被跳过，不会影响其他节点

## 🐛 故障排查

### 采集失败

1. 检查采集源 URL 是否可访问
2. 查看采集日志了解具体错误
3. 确认网络连接正常

### 节点未显示

1. 检查节点是否被关键词过滤
2. 确认节点协议是否支持
3. 查看采集日志确认节点是否被成功解析

### 定时采集不工作

1. 确认 Cron 任务是否正常运行
2. 检查是否启用了定时采集
3. 查看 Cron 日志确认任务是否执行
