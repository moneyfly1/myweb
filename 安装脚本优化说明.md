# 安装脚本优化说明

## ✨ 新增功能

### 1. 软件检测功能

安装脚本现在会自动检测已安装的软件，如果已安装则跳过安装步骤：

- ✅ **Nginx** - 检测是否已安装，如果已安装则跳过
- ✅ **PHP 8.2+** - 检测版本，如果版本符合要求则跳过
- ✅ **MariaDB/MySQL** - 检测是否已安装，如果已安装则跳过
- ✅ **Redis** - 检测是否已安装，如果已安装则跳过
- ✅ **Composer** - 检测是否已安装，如果已安装则跳过

### 2. 服务状态检测

脚本会检测服务是否正在运行：
- 如果服务已安装但未运行，会自动启动
- 如果服务正在运行，会显示确认信息

### 3. 数据库检测

- 检测数据库 `sspanel` 是否已存在
- 如果已存在，会询问是否使用现有数据库
- 如果不存在，会自动创建

## 🎯 使用场景

### 场景 1: 全新安装
- 所有软件都会自动安装
- 自动创建数据库和用户

### 场景 2: 部分软件已安装
- 已安装的软件会跳过
- 只安装缺失的软件
- 节省安装时间

### 场景 3: 所有软件已安装
- 所有安装步骤都会跳过
- 直接进入配置阶段
- 快速完成部署

## 📋 检测逻辑

### Nginx 检测
```bash
# 检测命令是否存在
command -v nginx

# 检测服务是否运行
systemctl is-active nginx
```

### PHP 检测
```bash
# 检测 PHP 版本
php -v

# 检查版本是否 >= 8.2
# 如果版本符合要求，跳过安装
```

### MariaDB/MySQL 检测
```bash
# 检测命令是否存在
command -v mariadb || command -v mysql

# 检测服务是否运行
systemctl is-active mariadb || systemctl is-active mysql
```

### Redis 检测
```bash
# 检测命令是否存在
command -v redis-server || command -v redis-cli

# 检测服务是否运行
systemctl is-active redis-server || systemctl is-active redis
```

### Composer 检测
```bash
# 检测命令是否存在
command -v composer
```

## 🔄 更新后的安装流程

1. **检测系统** - 自动检测操作系统
2. **安装基础工具** - 安装 curl, wget, git 等
3. **检测并安装 Nginx** - 如果未安装则安装
4. **检测并安装 PHP** - 如果版本不符合要求则安装/升级
5. **检测并安装 MariaDB** - 如果未安装则安装
6. **检测并安装 Redis** - 如果未安装则安装
7. **检测并安装 Composer** - 如果未安装则安装
8. **部署项目** - 从 GitHub 克隆项目
9. **配置项目** - 自动配置
10. **初始化数据库** - 如果数据库不存在则创建

## 💡 优势

1. **节省时间** - 跳过已安装的软件
2. **避免冲突** - 不会重复安装
3. **智能检测** - 自动判断是否需要安装
4. **灵活部署** - 支持部分安装和全新安装

## 🚀 使用方法

在 VPS 上运行：

```bash
wget https://raw.githubusercontent.com/moneyfly1/myweb/main/install.sh
chmod +x install.sh
sudo ./install.sh
```

脚本会自动检测已安装的软件并跳过安装步骤。
