# 后台访问问题解决指南

## 问题描述

安装完成后，访问后台出现 **404 Not Found** 错误。

## 快速修复步骤

### 步骤 1: 运行修复脚本

```bash
# 下载修复脚本
cd /var/www/sspanel
wget https://raw.githubusercontent.com/moneyfly1/myweb/main/fix_backend_access.sh
chmod +x fix_backend_access.sh
sudo ./fix_backend_access.sh
```

### 步骤 2: 创建管理员账号

```bash
# 下载创建管理员脚本
wget https://raw.githubusercontent.com/moneyfly1/myweb/main/create_admin.sh
chmod +x create_admin.sh
sudo ./create_admin.sh
```

脚本会提示您输入：
- 管理员邮箱
- 管理员密码
- 确认密码

## 手动修复方法

### 1. 检查 Nginx 配置

```bash
# 查找 Nginx 配置文件
ls -la /etc/nginx/conf.d/sspanel.conf
# 或
ls -la /www/server/nginx/conf/vhost/board.moneyfly.club.conf
```

### 2. 检查配置文件内容

配置文件应该包含：

```nginx
server {
    listen 80;
    server_name board.moneyfly.club;
    
    root /var/www/sspanel/public;  # 注意是 public 目录
    index index.php;
    
    location / {
        try_files $uri /index.php?$query_string;
    }
    
    location ~ \.php$ {
        fastcgi_pass unix:/tmp/php-cgi-82.sock;  # 根据实际 PHP 版本调整
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

**重要**: `root` 必须指向 `/var/www/sspanel/public`，不是 `/var/www/sspanel`！

### 3. 如果使用宝塔面板

1. 登录宝塔面板
2. 进入 **网站** → 找到 `board.moneyfly.club`
3. 点击 **设置** → **网站目录**
4. 确保目录是：`/var/www/sspanel/public`
5. 点击 **保存**

### 4. 检查项目目录结构

```bash
ls -la /var/www/sspanel/public
```

应该看到：
- `index.php`
- `.htaccess`（如果有）
- 其他文件

### 5. 检查文件权限

```bash
# 设置正确的权限
chown -R www-data:www-data /var/www/sspanel
chmod -R 755 /var/www/sspanel
chmod -R 777 /var/www/sspanel/storage
```

### 6. 检查 PHP-FPM

```bash
# 检查 PHP-FPM 是否运行
ps aux | grep php-fpm

# 检查 PHP socket
ls -la /tmp/php-cgi-*.sock
# 或
ls -la /run/php/php*-fpm.sock
```

### 7. 重启服务

```bash
# 重启 Nginx
systemctl restart nginx
# 或
/etc/init.d/nginx restart

# 重启 PHP-FPM（根据版本调整）
/etc/init.d/php-fpm-82 restart
# 或
systemctl restart php8.2-fpm
```

## 常见问题

### Q: 仍然显示 404

**A:** 检查以下几点：
1. Nginx 配置中的 `root` 是否指向 `public` 目录
2. `/var/www/sspanel/public/index.php` 是否存在
3. PHP-FPM 是否正常运行
4. Nginx 和 PHP-FPM 的 socket 路径是否匹配

### Q: 显示 500 错误

**A:** 通常是权限问题或 PHP 错误：
1. 检查文件权限
2. 查看 PHP 错误日志
3. 检查 `.config.php` 配置是否正确

### Q: 显示 "数据库连接失败"

**A:** 检查数据库配置：
1. 检查 `config/.config.php` 中的数据库配置
2. 确认数据库服务正在运行
3. 确认数据库用户和密码正确

### Q: 忘记管理员密码

**A:** 重置管理员密码：

```bash
cd /var/www/sspanel
php xcat Tool resetPassword
```

## 如果仍然无法访问

1. **检查防火墙**:
   ```bash
   ufw status
   # 或
   iptables -L
   ```

2. **检查域名解析**:
   ```bash
   ping board.moneyfly.club
   nslookup board.moneyfly.club
   ```

3. **检查端口**:
   ```bash
   netstat -tlnp | grep :80
   ```

4. **查看完整日志**:
   ```bash
   tail -100 /var/log/nginx/error.log
   tail -100 /var/log/nginx/access.log
   ```
