# SSPanel-UIM 节点采集改造大纲

## 项目背景
- **原项目 (cboard)**: 通过从外部 URL 采集节点，解析后生成 Clash 配置文件，节点数据从配置文件中读取
- **目标项目 (SSPanel-UIM)**: 当前通过手动配置服务器来添加节点，节点存储在数据库 `node` 表中
- **改造目标**: 将 SSPanel-UIM 的节点获取方式改为从外部 URL 采集，而不是手动配置

---

## 一、核心功能模块改造

### 1.1 节点采集服务 (新增)
**位置**: `src/Services/NodeCollector/`
- **NodeFetcher.php**: 从外部 URL 下载节点数据
  - 支持多个 URL 源
  - 支持 Base64 解码
  - 支持超时和重试机制
- **NodeParser.php**: 解析各种协议的节点链接
  - 支持协议: vmess, ss, ssr, trojan, vless, hysteria2, tuic
  - 将节点链接解析为结构化数据
- **NodeCollectorService.php**: 协调采集流程
  - 调用 NodeFetcher 获取节点
  - 调用 NodeParser 解析节点
  - 将解析后的节点保存到数据库

**参考代码**: `/Users/apple/Downloads/cboard 网站/app/services/config_update_service.py` 中的 `NodeFetcher` 和 `NodeParser` 类

---

### 1.2 节点模型改造
**位置**: `src/Models/Node.php`
- **保持现有字段**: id, name, server, type, sort, traffic_rate 等
- **新增字段** (可选):
  - `source_url`: 节点来源 URL
  - `collected_at`: 采集时间
  - `is_collected`: 是否为采集节点 (区分手动配置和采集节点)
- **修改逻辑**: 
  - 节点数据从采集服务自动填充，而不是手动输入
  - 保留手动配置功能作为备选

### 1.3 节点管理控制器改造
**位置**: `src/Controllers/Admin/NodeController.php`

#### 1.3.1 新增功能
- **采集配置页面**: 配置节点采集源 URL
  - 添加/删除采集源
  - 设置采集间隔
  - 启用/禁用自动采集
- **手动触发采集**: 立即执行一次节点采集
- **采集日志查看**: 查看采集历史记录

#### 1.3.2 修改功能

- **节点编辑**: 采集节点可编辑部分字段（如 name, traffic_rate），但 server 等核心字段建议只读
- **节点删除**: 采集节点删除后，下次采集可能重新出现（需要标记机制）

### 1.4 订阅服务改造
**位置**: `src/Services/Subscribe.php` 和 `src/Services/Subscribe/*.php`

#### 1.4.1 修改 `Subscribe::getUserNodes()`
- **当前逻辑**: 从数据库 `node` 表查询节点

## 九、注意事项

### 9.1 数据一致性
- 采集节点可能会重复（相同 server+port）
- 需要去重机制：根据 server+port+协议 判断是否已存在
- 更新已存在节点，而不是创建新节点

### 9.2 节点更新策略
- **全量更新**: 每次采集清空所有采集节点，重新导入（简单但可能丢失手动修改）
- **增量更新**: 只添加新节点，更新已存在节点（推荐）
- **标记删除**: 标记未采集到的节点为失效，但保留在数据库中

### 9.3 性能考虑
- 采集可能耗时较长，使用异步任务或后台任务
- 大量节点时，批量插入数据库
- 添加缓存机制，避免频繁采集

### 9.4 错误处理
- URL 访问失败的处理
- 节点解析失败的处理
- 数据库操作失败的回滚

## 总结

本次改造的核心是将 SSPanel-UIM 从"手动配置节点"改为"自动采集节点"，同时保留手动配置功能作为备选。主要改动包括：

1. **新增节点采集服务**（最重要）
2. **修改节点管理界面**
3. **添加定时采集任务**
4. **数据库结构扩展**
5. **保持订阅功能兼容**

改造后，系统将能够自动从外部 URL 采集节点，大大减少手动配置的工作量。
